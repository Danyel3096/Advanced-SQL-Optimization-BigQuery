# 🚀 Analítica Avanzada y Optimización de Consultas con SQL & BigQuery

## 🎯 Objetivo del Proyecto
Este repositorio demuestra el dominio de técnicas avanzadas de SQL para el manejo de Big Data. A diferencia del análisis básico, aquí el enfoque se centra en la **eficiencia computacional (costos)** y el uso de **funciones analíticas** para resolver problemas complejos de negocio, como el análisis de tendencias temporales y la jerarquización de datos.

## 🛠️ Herramientas Utilizadas
* **Plataforma Cloud:** Google BigQuery.
* **Lenguajes:** SQL Avanzado (Standard SQL), Python.
* **Datasets:** Stack Overflow, San Francisco Open Data, y datasets de e-commerce.

## 🚀 Habilidades Técnicas Demostradas (Nivel Avanzado)
En este nivel, he implementado soluciones que van más allá del CRUD básico:

1. **Window Functions (Funciones de Ventana):** Uso de `OVER`, `PARTITION BY` y `RANK` para realizar cálculos complejos a través de conjuntos de filas relacionados sin colapsar las filas (vital para análisis de rankings y series de tiempo).
2. **Optimización de Costos y Rendimiento:** Aplicación de estrategias para reducir la cantidad de datos escaneados en BigQuery, optimizando el uso de la cuota de la API y mejorando la velocidad de respuesta.
3. **Manejo de Datos Complejos (Nested & Repeated Fields):** Capacidad para consultar estructuras de datos no relacionales dentro de SQL (como JSON o Arrays), una habilidad crítica en entornos modernos de Data Engineering.
4. **Analítica de Series Temporales:** Implementación de funciones `LEAD` y `LAG` para comparar valores entre periodos y detectar tendencias de crecimiento o caída.
5. **Joins Complejos y CTEs de Alto Rendimiento:** Estructuración de consultas multi-nivel para procesos ETL eficientes.
